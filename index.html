HTML4 vs. HTML5

HTML5 brings a lot of improvements over HTML4. It adds in more accessibilty features, improves audio/video capabilities, provides cleaner, and better laid out code, and uses client-side storage better. The following is a summary of those changes.

New Elements

Semantic Elements

HTML5 brings new elements that are cleaner and easier to read (by search engines, screen readers, and humans) and easier to write and edit (sections are easier to identify than divs, for example).

The following new semantic elements are used for sectioning content:

  • section is used for a generic section of a document.
  • article is most widely used for blog entries and news articles.
  • main is used to demark the main content of a document.
  • aside is used for a section of content that is loosely related to the main content of the page. An example would be a box of related links or some fun trivia.
  • header is used for a page's introduction (title, logo, main navigation section).
  • footer is used for bottom of the page content (items like copyright information, links to author and contact information).
  • nav is used for the main navigation of the site.
  • figure is used for self-contained flow content.
  • figcaption an optional caption element used with figure.
  • template is used to create fragments of HTML that can scripted to be cloned and re-used in the document.

Previously, DIVs were used for layout:

<div id="header">
<h1>Header Text</h1>
  <div id="nav">
    <ul>
      <li>link1</li>
      <li>link2</li>
      <li>link3</li>
    </ul>
  </div>
</div>

A cleaner layout using HTML5 sectioning tags:

<header>
<h1>Header Text</h1>
  <nav>
    <ul>
      <li>link1</li>
      <li>link2</li>
      <li>link3</li>
    </ul>
  </nav>
</header>

Other new HTML5 elements

  • new attributes for forms (number, date, time, calendar, and range)
  • graphic elements (svg and canvas)
  • multimedia elements (audio and video)

Example of using the video element, without the need to use a plug-in:

  <video width="320" height="240" controls>
    <source src="movie.mp4" type="video/mp4">
    <source src="movie.ogg" type="video/ogg">
  Your browser does not support the video tag.
  </video>

Create your own elements in HTML5

<!doctype html>
<html>
  <head>
    <style>
      small {
      display: block;
      background-color: black;
      padding: 2rem;
      font: 11px;
      }
    </style>
  </head>
  <body>
  <h1>My Heading</h1>
    <small>Small Element</small>
  </body>
</html>

Coding Conventions

HTML5 isn't strict about code validation but adhering as much to "XHTML-style" syntax can future-proof your HTML.

Doctype

HTML4 Transitional DOCTYPE:

  <!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01 Transitional//EN">
  "http://www.w3.org/TR/html4/loose.dtd"

HTML5 DTD-less DOCTYPE:

  <!doctype html>

Encoding

HTML4 encoding:

  <meta http-equiv="Content-Type" content="text/html;charset=utf-8">

HTML5 encoding:

  <meta charset="utf-8">

Lower case element names

While HTML5 allows the use of uppercase and lower case letters in element names, for consistency, you should use all lower case letters. Your code will look cleaner and be easier to write.

Bad

  <HEADER>
  <H1>Header Text</H1>
    <p>Some sentence.</p>
  </HEADER>

Good

  <header>
  <h1>Header Text</h1>
    <p>Some sentence.</p>
  </header>

Other coding conventions include:

  • Close empty html elements.
  • Use lower case attribute names.
  • Quote attribute values.
  • Include alt attribute with images.
  • Remove spaces around equal signs.

New HTML5 Application Programming Interfaces (APIs)

Highlights include:

  • HTML Geolocation: Used to find a user's position
  • HTML Drag and Drop: Make an element draggable (and droppable)
  • HTML Local Storage: Store data locally, in the user's browser
  • HTML Web Workers: JavaScript runs in the background to improve performance
  • HTML Server-Sent Events: Web page automatically gets updates from a server
